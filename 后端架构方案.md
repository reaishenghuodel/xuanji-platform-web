## 二、技术栈选型

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| 后端框架 | FastAPI + Python 3.12 | 异步高性能、自动文档、类型提示、社区活跃 |
| 数据库 | PostgreSQL 16 | 成熟稳定、JSONB 支持半结构化数据、全文检索 |
| 缓存 | Redis 7 | 会话缓存、限频、排行榜、实时数据 |
| 时序数据 | ClickHouse 24 | 海量监测数据高效写入和聚合查询 |
| 对象存储 | MinIO | S3 兼容、报告文件/图片存储 |
| 队列 | Celery + Redis | 异步任务（数据爬取、报告生成、通知发送） |
| 监控 | Prometheus + Grafana | 系统指标、业务指标监控 |
| 日志 | ELK (Elasticsearch + Logstash + Kibana) | 日志聚合、搜索、可视化 |
| 部署 | Docker + Docker Compose | 开发环境快速启动，后续可迁移到 K8s |

## 三、数据库设计（核心表）

### 3.1 用户与权限

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    name VARCHAR(100) NOT NULL,
    avatar_url VARCHAR(500),
    phone VARCHAR(20),
    company_name VARCHAR(200),
    role VARCHAR(20) DEFAULT 'user' CHECK (role IN ('user', 'admin', 'superadmin')),
    plan_type VARCHAR(20) DEFAULT 'free' CHECK (plan_type IN ('free', 'pro', 'enterprise')),
    plan_expire_at TIMESTAMP,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 团队表
CREATE TABLE teams (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(200) NOT NULL,
    owner_id UUID REFERENCES users(id),
    logo_url VARCHAR(500),
    settings JSONB DEFAULT '{}',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 团队成员
CREATE TABLE team_members (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    team_id UUID REFERENCES teams(id) ON DELETE CASCADE,
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    role VARCHAR(20) DEFAULT 'member' CHECK (role IN ('owner', 'admin', 'member')),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(team_id, user_id)
);
```

### 3.2 品牌与监测

```sql
-- 品牌表

    neutral_sentiment DECIMAL(5,2),
    negative_sentiment DECIMAL(5,2),
    accuracy_score DECIMAL(5,2),
    citation_count INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

-- 引用详情

    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
```

## 六、部署架构

### 6.1 Docker Compose（开发环境）

```yaml
# docker-compose.yml
version: '3.8'

services:
  api:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql://user:pass@postgres:5432/xuanji
      - REDIS_URL=redis://redis:6379/0
      - DEEPSEEK_API_KEY=${DEEPSEEK_API_KEY}
    depends_on:
      - postgres
      - redis
      - clickhouse

  postgres:
    image: postgres:16-alpine
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=xuanji
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=pass

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data

  clickhouse:
    image: clickhouse/clickhouse-server:24
    volumes:
      - clickhouse_data:/var/lib/clickhouse

  minio:
    image: minio/minio:latest
    command: server /data --console-address ":9001"
    volumes:
      - minio_data:/data

  celery:
    build: ./backend
    command: celery -A app.celery worker --loglevel=info
    depends_on:
      - redis
      - postgres

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./dist:/usr/share/nginx/html
```

## 十、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| AI 平台 API 不稳定 | 监测数据缺失 | 多源采集 + 本地缓存 + 兜底机制 |
| 爬虫被反爬 | 数据采集受阻 | 分布式 IP 池 + 请求频率控制 + 模拟行为 |
| 数据量爆炸 | 存储成本激增 | ClickHouse 分片 + 数据归档策略 |
| 并发过高 | 系统性能下降 | 缓存 + 限频 + 队列削峰 + 水平扩展 |
| 安全漏洞 | 数据泄露 | 输入校验 + SQL 注入防护 + XSS 过滤 + 审计日志 |
